## Step 4：Phase 2 — Token Scoring（改进的 Delta 计算） 目标：对 D_train 全集（~13300 训练样本）做 token-level 评分，确保 100% 覆盖。核心改进：不再
使用裸减法，改用分阶段的改进评分方案。
### 核心变更：
- 旧流程：Phase 2 只评分了 D_ref 的 train split（807/1000），导致 193 样本无 mask 回退到 full training
- 新流程：Phase 2 评分 D_train 全集（~13300 样本），保证每个训练样本都有 mask
- Delta 计算升级：从裸减法 `ℓ_target - ℓ_ref` 升级为标准化 excess loss + 软权重（详见 `tts_token_selection_design.md`）
### Delta 计算改进方案（分阶段实施）：
详细的理论分析和方案推导见：prompts and Report/tts_token_selection_design.md 阶段 1（本次实验采用）：标准化 excess loss
为什么不用裸减法：
- 尺度敏感问题：reference 与 target 的 loss 不一定在同一尺度上，训练阶段、模型 calibration、概 率分布熵水平不同都会导致差值含义不一致
- 高 loss ≠ 高价值：语音中高 loss token 可能来自数据噪声、tokenizer 离散化误差、对齐困难等， 并非都是"值得学习"的 token
- batch 间漂移：不同 batch 的 loss 分布可能差异很大，裸减法的 top-k 排序不够稳定 采用方案：batch 内 z-score 标准化差值（方案 A2）
设当前 batch 中 target loss 均值/标准差为 μ_t, σ_t，reference loss 均值/标准差为 μ_r, σ_r：

z_t(i) = (ℓ_target,i - μ_t) / (σ_t + ε) z_r(i) = (ℓ_ref,i - μ_r) / (σ_r + ε) s_i = z_t(i) - z_r(i)
优点：
- 消除不同 batch 间的尺度漂移
- 直观含义："该 token 相对 target 更难、相对 reference 更异常"
- 与现有"每 batch 排序、取前 60%"最兼容
- 能直接缓解"delta 退化为 0 或分布很窄"的问题 阶段 2（可选升级）：软权重替代硬筛选 在标准化分数基础上，不做硬 top-k 截断，改用 sigmoid 映射为连续权重：
代码块 w_i = σ(α · s_i) 训练损失变为：
代码块 L = Σ_i w_i · ℓ_target,i / (Σ_i w_i + ε) 理由：语音信号具有连续性，硬截断可能导致训练信号不连续，软权重更平滑稳定。
阶段 3（长远方向）：引入 reward / KL 混合打分
代码块 s_i = α · norm(ℓ_target,i - ℓ_ref,i) + β · norm(ℓ_target,i) + γ · r_i

结合 ASR reward、SER reward、speaker similarity 等下游信号，使 token 选择与 TTS 真正目标对 齐。加入 KL 正则防止模型过度偏离 reference。
